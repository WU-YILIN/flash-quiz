# 01-02 Plan: Add Bootstrap Registrations and Ribbon Entry

## Objective

Implement the first working plugin bootstrap with a ribbon icon and a minimal user-visible action.

## Scope

- implement plugin class bootstrap
- register ribbon icon
- register a placeholder command or notice path
- ensure lifecycle methods are structured for later extension

## Tasks

1. Implement the main plugin class and `onload` / `onunload`.
2. Add a ribbon icon that opens a placeholder modal or notice.
3. Add one internal service boundary for later modules, even if still simple.
4. Keep startup logic minimal and defer heavier work.

## Must Haves

- Plugin loads without runtime errors
- Ribbon icon is visible in Obsidian
- Clicking the ribbon triggers a visible response
- Code structure leaves room for settings, question bank, and modal modules

## Verification

- Plugin can be enabled in a development vault
- Ribbon icon click works consistently
- No heavy work runs during startup

## Risks

- Letting bootstrap code absorb responsibilities that belong to later modules
- Tying the first visible action too tightly to a future modal implementation
