# 01-03 Plan: Add Baseline Styles and Development Notes

## Objective

Prepare the plugin for iterative UI work by adding base styles and clear local development instructions.

## Scope

- add plugin-scoped baseline styles
- avoid hardcoded global overrides
- document how to run, build, and load the plugin during development

## Tasks

1. Create a small plugin-scoped CSS baseline for later modal and settings work.
2. Add naming and class conventions that avoid clashing with Obsidian core styles.
3. Document development workflow in `README.md`.
4. Document how to symlink or copy build outputs into an Obsidian test vault.

## Must Haves

- Styles are namespaced to the plugin
- Documentation is enough to reproduce local development
- Notes align with official Obsidian plugin development flow

## Verification

- CSS loads without affecting unrelated Obsidian UI
- Another developer can follow README steps to start the plugin locally

## Risks

- Over-designing CSS before actual UI modules exist
- Leaving setup instructions too implicit for repeated local testing
